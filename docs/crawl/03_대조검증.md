# `pinterest img crawlling.md` (v2) 대조·검증 정리

작성: 2026-07-07 · 기준 구현: `composition_dataset/`

v2 요구사항 문서를 현재 구현과 항목별로 대조하고, 문서만으로 판단이 안 되는 부분은
실제로 실측·검증한 결과를 정리한다. **결론: 핵심은 대부분 (다른 도구로) 이미 해결됐고,
진짜 새로 참고할 가치가 있는 건 4가지다.**

## 1. 이미 해결된 것 (실측 확인 포함)

- **원본 해상도**: 문서는 pinterest-dl(API)을 써야 736px 상한을 넘는다고 했다. 나는 gallery-dl을
  쓰는데, 이것도 Pinterest 내부 API 기반이라 동일하게 원본을 받는다. **실측 결과 78%가 736px 초과,
  최대 6000px** — 문서가 걱정한 문제를 이미 회피했다.
- **검수 갤러리**: 문서는 카테고리별 개별 HTML(approve/reject 체크박스)을 요구했다. 내 `gallery.py`는
  한 파일에서 4축 33코드를 **조합 필터**(카페 AND 앉기 AND 로우앵글)할 수 있어 문서 요구를 넘어선다.
- **감성 필터**: 문서는 검색어에 `korean/aesthetic/snap` 같은 단어를 넣어 스톡 느낌을 줄이는 데
  그친다. 나는 그 위에 **CLIP으로 한국인·인스타 감성 점수를 매겨 외국/스톡을 자동 rejected** 처리한다
  (승인분 인스타 0.97 / 한국인 0.94). 문서에 없는 강화 기능이다.

## 2. 새로 참고할 가치가 있는 것 (채택 후보, 가치순)

### ★ (1) 시드핀 연관핀 수집 — 방식 B
문서의 가장 유용한 아이디어. 검색은 뒤로 갈수록 관련도가 떨어지지만, 마음에 드는 대표 핀의
**연관핀(Pinterest 추천엔진)** 은 구도·무드 적합도가 높다. 특히 지금 **희소한 축**
(점프 4, 뒷모습 6, 상반신클로즈업 0, 로우/하이앵글 한 자릿수) 보강에 최적.
→ **gallery-dl `related-pin` extractor 지원 확인됨.** 시드 핀 URL만 있으면 바로 구현 가능.

### ★ (2) 로그인 쿠키 세션
문서: 비로그인은 검색어당 25~50장에서 로그인 벽 → 검색어당 300+ 안정 확보엔 쿠키 필수.
내 현재 수집은 검색어당 ~14장(카테고리 100 ÷ 7검색어) 수준이라 아직 벽에 안 부딪혔지만,
**대량 확장하면 반드시 걸린다**. `pinterest.py`가 `GALLERY_DL_COOKIES`를 이미 받으므로,
쿠키 추출 절차만 README에 넣고 쓰면 된다. (계정 제한 리스크는 감수 — 문서 15장)

### (3) final_score 랭킹 + YOLO person detector
문서: `2.0*clip + person + resolution + aspect + keyword`로 검수 큐를 정렬하고,
YOLO로 사람 유무를 판정(내 CLIP person_visible보다 견고). 팀 열람 목적에선 "상위부터 보면 됨"이
검수 속도를 올린다. 중간 가치 — 지금은 quality_score만 있어 랭킹이 거칠다.

### (4) URL 정규화 사전 dedup
문서: pinimg URL의 크기 프리픽스를 떼고 `sha1(정규화경로)[:12]`를 image_id로 삼아,
**다운로드 전에** 리핀 중복을 제거 → 대역폭 절약 + 결정적 resume. 나는 받은 뒤 pHash로 잡는다
(지난 실행에서 307장 중복 제거 — 상당수는 사전 dedup으로 안 받을 수 있었다). 현재 규모에선
낮은 우선순위지만 1만 장 규모면 의미 있다.

## 3. 설계상 다르게 해결한 것 (결함 아님)

- **글로벌 dedup vs 카테고리내 dedup**: 문서는 4축 폴더로 각각 수집하므로 "cafe 검색"과 "sitting
  검색"에서 같은 사진이 두 폴더에 잡힌다 → 그래서 카테고리 **내부만** dedup하고 교차축은 멀티라벨로
  보존해야 한다(문서 결함4). 반면 **나는 WHERE로만 검색하고 나머지 3축은 CLIP이 라벨링**하므로
  같은 사진이 여러 폴더에 생기는 상황 자체가 없다. 이미지 1벌 + CSV 멀티라벨이라 **글로벌 dedup이
  오히려 정답**이다. 즉 문서가 공들여 푼 문제를 내 설계는 우회한다.
- **폴더 구조**: 문서는 `00_raw/{axis}/{category}`. 나는 원본 `image crolling.md`(v1) 구조
  (raw→curated→processed + master_metadata.csv)를 따른다. 둘 다 유효.

## 4. 열린 질문 (진행 전 결정 필요)

1. **도구: gallery-dl 유지 vs pinterest-dl 전환?**
   → 검증 결과 gallery-dl이 원본해상도·search·related-pin·쿠키를 다 지원. **전환 실익 없음, 유지 권장.**
   (pinterest-dl로 바꾸면 sources/pinterest.py 재작성 필요 — 이득 대비 비용 큼)
2. **목적 범위**: v2 문서는 목적을 "**팀 열람용 구도 레퍼런스**(학습 데이터 아님)"로 좁혔다.
   그렇다면 내 파이프라인의 train/val/test split·다중해상도 리사이즈는 당장 불필요.
   → **지금 목표가 "열람용 대량 레퍼런스"인가, "학습 데이터"인가?** (이게 우선순위를 가른다)
3. **쿠키 세션 도입?** 대량 확장(검색어당 300+)하려면 필요. 계정 제한 리스크 감수 여부.
4. **희소 축 보강 방법**: 방식 A(포즈/앵글 검색어 boost, 이미 queries.yaml에 있음) vs
   방식 B(시드핀 연관핀). B가 관련도 높지만 사람이 시드 핀을 골라야 함(카테고리당 5~10분).
5. **YOLO/final_score 랭킹 도입?** 검수량이 많아지면 가치↑, 지금 규모(386장)면 갤러리 필터로 충분.

## 5. 추천 다음 액션 (우선순위)

1. **목적부터 확정** (열린질문 2) — 열람용이면 split/resize 스킵하고 수집·필터·갤러리에 집중.
2. **희소 축 보강** — 즉시 실행 가능한 방식 A(포즈/앵글 boost 검색어)로 점프·뒷모습·로우앵글 등부터.
   더 순도가 필요하면 방식 B(시드핀) 추가.
3. **대량 확장이 목표라면 쿠키 세션 + (선택) URL 정규화 사전 dedup** 도입.
4. **검수량이 커지면 final_score 랭킹** 추가.

> 요약: v2 문서의 "실측 가능하게 만든" 핵심 전제(원본 URL, 검증된 도구, 멀티라벨, 검수 자동화)는
> 이미 충족됐거나 더 나은 방식으로 해결돼 있다. 새로 가져올 것은 **시드핀 연관핀·쿠키 세션·
> 점수 랭킹·사전 dedup** 4가지이며, 이 중 무엇을 할지는 위 "열린 질문"의 목적 확정에 달렸다.

## 6. 결정 및 구현 결과 (2026-07-07)

**목적**: "열람 먼저, 학습 나중" — 지금은 열람용 대량 레퍼런스에 집중, split/resize 유지하되 부차.
**이번에 적용**: (1) 시드핀 연관핀(방식 B), (2) final_score 랭킹 + YOLO. (쿠키·사전 dedup은 보류)

| 구현 | 파일 | 검증 |
|---|---|---|
| 시드핀 연관핀 수집 | `scripts/collect/seeds.py` + `sources/pinterest.py::collect_related` | 실제 핀 1개→연관 12장 수집 성공. `#related` 트리거 |
| 핀 출처 URL 수정 | `sources/pinterest.py::_read_source_url` | JSON `id`로 `pinterest.com/pin/<id>/` 구성 (기존엔 이미지 CDN URL만 저장되던 버그 수정). 작가명도 `pinner`에서 추출 |
| final_score 랭킹 | `scripts/rank/rank.py` → `annotations/review_queue.csv` | 386장 점수화, final -6~11, YOLO 사람검출(1인 269/다인 88/미검출 29), category_match=False 128장 자동 플래그 |

- 시드 입력: `config/seed_pins.csv` (axis, category_id, seed_pin_url, related_count). 주석/`/pin/` 없는 행은 무시.
- 점수식은 문서 10장 그대로. torch/open_clip/ultralytics 미설치 시 해당 항 0으로 graceful degradation.
- **도구 유지**: gallery-dl이 원본해상도·search·related·쿠키를 다 지원하므로 pinterest-dl 전환 안 함.

**보류(필요 시 추가)**: 로그인 쿠키 세션(대량 확장 시), URL 정규화 사전 dedup(1만장 규모 시).
