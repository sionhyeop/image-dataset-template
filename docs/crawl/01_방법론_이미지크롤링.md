좋은 방향이야. 다만 **구도/장소/포즈/무드 이미지는 한 이미지가 여러 카테고리에 동시에 속하는 경우가 많기 때문에**, 폴더를 너무 깊게 나누면 나중에 학습 데이터 관리가 어려워져.

예를 들어 한 장의 사진이 동시에:

> 카페 + 앉기 + 반신 + 아이레벨 + 자연광 + 소품 활용

일 수 있어.
그래서 추천 방식은 **이미지 파일은 평평하게 관리하고, 카테고리는 CSV/JSON 메타데이터로 관리하는 구조**야.

# 2. 절대 비추천하는 구조

처음 보면 이런 식으로 나누고 싶을 수 있어.

```txt
카페/
└─ 앉기/
   └─ 반신/
      └─ 아이레벨/
         └─ 자연광/
            └─ image001.jpg
```

이 구조는 나중에 거의 망가져.

왜냐하면 이미지 한 장이 여러 라벨을 동시에 가질 수 있기 때문이야.
예를 들어 같은 사진이 `카페`, `앉기`, `소품 활용`, `반신`, `아이레벨`, `필카 감성`에 전부 해당될 수 있는데, 폴더 하나에만 넣으면 정보가 손실돼.

그래서 **최종 데이터셋은 폴더보다 메타데이터 중심으로 관리**하는 게 좋아.

# 5. 이미지 파일명 규칙

파일명은 카테고리명을 다 넣기보다 **고유 ID 중심**으로 가는 게 좋아.

```txt
img_000001.jpg
img_000002.jpg
img_000003.jpg
```

왜냐하면 파일명에 모든 정보를 넣으면 너무 길어져.

```txt
cafe_sitting_halfbody_eyelevel_naturallight_00001.jpg
```

이렇게 하면 처음엔 좋아 보이지만, 나중에 라벨을 수정할 때 파일명도 계속 바꿔야 해.
그래서 파일명은 단순하게 두고, 라벨은 CSV에서 관리하는 게 좋아.

# 7. 크롤링 단계 폴더 구조

크롤링 원본은 일단 검색어/출처 기준으로 보관해도 돼.

```txt
data/00_raw/
└─ 2026-07-06/
   ├─ google/
   │  ├─ W01_cafe/
   │  ├─ W02_beach_sea/
   │  ├─ P01_sitting/

   │
   ├─ pinterest/
   │  ├─ W01_cafe/
   │  ├─ F01_full_body/

   │
   └─ unsplash/
      ├─ W02_beach_sea/
      └─ S06_natural_light_beach/
```

단, 이건 **원본 수집용 폴더**일 뿐이야.
학습에 바로 쓰는 폴더가 아니야.

크롤링할 때는 검색어도 같이 저장하는 게 좋아.

예시:

```csv
raw_file,source,search_query,source_url,downloaded_at
raw_000001.jpg,pinterest,"cafe sitting girl portrait",https://...,2026-07-06
raw_000002.jpg,google,"beach walking full body portrait",https://...,2026-07-06
```

# 8. 검수 후 curated로 옮기는 방식

크롤링한 이미지는 바로 학습에 쓰면 안 돼.
아래 기준으로 한 번 걸러야 해.

```txt
사용 가능:
- 사람이 명확히 보임
- 구도 라벨이 분명함
- 해상도 충분함
- 과도한 워터마크 없음
- 너무 흐리거나 깨지지 않음
- 중복 이미지 아님

제외:
- 저작권/출처 불명확
- 얼굴/몸이 너무 잘림
- 라벨이 애매함
- 너무 작은 이미지
- AI가 만든 티가 심한 이미지
- 워터마크가 큼
- 같은 이미지의 리사이즈/재업로드 버전
```

검수 결과는 이런 식으로 관리하면 좋아.

```txt
status:
- raw
- approved
- rejected
- needs_review
```

# 9. 학습용 split은 폴더 복사보다 CSV 추천

최종 학습 데이터는 이렇게 CSV로 나누는 게 좋아.

```txt
data/03_splits/
├─ train.csv
├─ val.csv
└─ test.csv
```

예를 들어 `train.csv`:

```csv
image_id,file_path,where,pose_action,shot_size,camera_style
img_000001,data/02_processed/768/img_000001.jpg,W01_cafe,P01_sitting,F02_half_body,"S02_eye_level;S05_film_mood"
img_000002,data/02_processed/768/img_000002.jpg,W02_beach_sea,P03_walking_snap,F01_full_body,"S06_natural_light_beach"
```

추천 비율은 보통:

```txt
train: 80%
val: 10%
test: 10%
```

또는 데이터가 적으면:

```txt
train: 70%
val: 15%
test: 15%
```

중요한 건 **같은 이미지의 변형본이 train과 test에 동시에 들어가면 안 된다**는 거야.
예를 들어 같은 사진의 크롭본, 리사이즈본, 흑백본이 train/test에 나뉘어 들어가면 성능이 실제보다 높게 나올 수 있어.

# 10. 카테고리별 최소 수량 기준

초기 MVP라면 소분류당 이 정도부터 시작해도 돼.

```txt
소분류당 최소 100~300장
괜찮은 수준: 500장 이상
제대로 학습: 1,000장 이상
```

하지만 단순히 카테고리별 장수만 맞추면 안 돼.

예를 들어 `카페` 이미지만 1,000장 있는데 대부분이 `앉기 + 반신 + 아이레벨`이면, 모델은 카페를 장소가 아니라 “앉아 있는 반신 인물 사진”으로 잘못 배울 수 있어.

그래서 조합도 어느 정도 분산해야 해.

예시:

```txt
W01_cafe + P01_sitting + F02_half_body
W01_cafe + P02_leaning + F01_full_body
W01_cafe + P07_using_props + F03_closeup
W01_cafe + P03_walking_snap + F01_full_body
```

이런 식으로 같은 장소 안에서도 포즈와 프레이밍을 다양하게 넣어야 좋아.

# 11. 실제 운영 방식 추천

## 1단계: 크롤링

```txt
data/00_raw/날짜/출처/검색쿼리/
```

에 저장.

## 2단계: 중복 제거

이미지 해시나 pHash로 거의 같은 이미지를 제거.

## 3단계: 1차 검수

너무 작은 이미지, 깨진 이미지, 워터마크 이미지, 이상한 이미지를 제거.

## 4단계: 라벨링

각 이미지에 아래 4축 라벨 부여.

```txt
WHERE
POSE_ACTION
SHOT_SIZE
CAMERA_STYLE
```

## 5단계: curated 이동

승인된 이미지만:

```txt
data/01_curated/images/
```

에 `img_000001.jpg` 형태로 저장.

## 6단계: 리사이즈/전처리

학습 모델에 맞게:

```txt
data/02_processed/512/
data/02_processed/768/
data/02_processed/1024/
```

에 저장.

## 7단계: train/val/test split 생성

```txt
data/03_splits/train.csv
data/03_splits/val.csv
data/03_splits/test.csv
```

생성.

# 14. 저작권/초상권 주의

웹크롤링 이미지를 AI 학습에 쓸 때는 꼭 주의해야 해.

특히 사람 얼굴이 들어간 사진은 **초상권, 저작권, 플랫폼 이용약관** 문제가 생길 수 있어.
개인 프로젝트 내부 실험이면 리스크가 낮을 수 있지만, 상용 서비스 학습 데이터로 쓰려면 출처, 라이선스, 사용 가능 범위를 반드시 관리해야 해.

그래서 `master_metadata.csv`에 아래 필드는 꼭 넣는 걸 추천해.

```csv
source_url,source,license,author,usage_allowed
```

최소한 나중에 “이 이미지 어디서 가져왔지?”를 추적할 수 있어야 해.

# 15. 너희 프로젝트에 맞는 최종 요약

너희는 “구도별 이미지 추천/학습”에 가까워 보이니까 폴더 구조는 이렇게 가는 게 제일 좋아.

```txt
composition_dataset/
├─ taxonomy.yaml
├─ data/
│  ├─ 00_raw/              # 크롤링 원본
│  ├─ 01_curated/images/   # 검수 완료 원본
│  ├─ 02_processed/        # 학습용 리사이즈 이미지
│  └─ 03_splits/           # train/val/test CSV
├─ annotations/
│  └─ master_metadata.csv  # 이미지별 다중 라벨
└─ scripts/
   ├─ crawl/
   ├─ deduplicate/
   ├─ resize/
   └─ make_splits/
```

그리고 카테고리는 폴더로 깊게 나누지 말고, 아래 4축 라벨로 관리하는 걸 추천해.

